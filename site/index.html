What you get

The CLI you know, with the layer it was missing.

Everything upstream flow does — plus the connectors, triage, and autonomy that make a browser worth leaving open all day.

One task, one session, one tab

flow do <task> spawns a dedicated tab — iTerm2, Warp, Terminal, kitty, Ghostty, or zellij. Tomorrow's flow do resumes the same conversation.

Claude or Codex, your call

Provider is per-task — pass --agent codex or keep Claude. The knowledge base, briefs, and close-out sweep work identically either way.

Worktrees by default

Each task gets a git worktree on flow/<slug>, so parallel work never collides. flow done pushes the branch and opens a PR with the brief as its body.

Mission Control in your browser

flow ui serve boots a local web app: task views, inline editing, a Cmd+K palette, and a browser-attached terminal streaming the live agent.

Attention Router

A cross-source triage queue that watches your connectors, scores what deserves a look, and surfaces cards you act on — surface-only until you opt into more.

Owners & autonomous runs

Durable, repo-scoped controllers that wake on a cadence to dispatch work — plus flow do --auto headless runs that work end-to-end and close themselves.

Slack & GitHub triggers

React to a thread with :claude:, or get assigned a PR — flow spins up a task bound to that source, with the same KB and UI.

Per-task model selection

Pin a model with --model, or let flow resolve a tier at launch — upshifted for high-priority work, downshifted for richly-specified briefs.

Full-text search & daily briefing

flow search over briefs, updates, KB, and transcripts via SQLite FTS5. flow standup assembles a copyable needs-action / FYI digest.

A knowledge base that grows

Five markdown buckets for durable facts about you, your team, products, processes, and customers — live-appended during sessions, swept from transcripts on done.

Two names, one tool. The CLI binary is flow — every command is flow do, flow ui serve, flow done; scripts and muscle memory from upstream flow keep working unchanged. The browser product — Mission Control — wears the flowwyyy name. New here? It's flow at the terminal and flowwyyy in the browser.

A tour

One consistent view — terminal, browser, and skill.

Same SQLite, same markdown briefs, same skill. Mission Control is a peer to the CLI, not a replacement.

Get started

Local, portable, yours.
